Changelog
Python-only stack, fitness_age fix, prompt extraction. arch
Replaced send-email.js (Node/Resend SDK) with send_email.py (Python stdlib urllib) — Node runtime fully removed. Extracted Claude prompt from run_recap.sh into prompt.md for easier iteration. Consolidated Garmin credentials from .garmin_config.json into .env.local. Fixed fitness_age API call (get_fitnessage → get_fitnessage_data). Emails now sent from market@joseandgoose.com. v1.2.0
OAuth token persistence — no more daily SSO logins. fix
garmin_fetch.py now saves garth OAuth tokens to .garth_tokens/ after the first credential login and reuses them on every subsequent run. Eliminates daily calls to Garmin's SSO embed endpoint, which was triggering 429 rate-limits after the original 5-retry loop hammered the endpoint 5× in 5 minutes. Exit code 2 added for 429 responses so run_recap.sh aborts immediately instead of retrying. Tokens seeded from Mac using a browser User-Agent to bypass the account-level SSO block. v1.1.0
Initial release — Garmin daily health recap automation. feat
Daily pipeline: garmin_fetch.py pulls 10 health metrics from Garmin Connect, garmin_slim.py condenses 250KB raw JSON to ~3KB, Claude CLI generates a structured recap (Focus Mode, Body Battery, Sleep, RHR, Stress, VO2 Max, Activity), send_email.py delivers via Resend to Gmail. Runs at 7am via cron; check-recap.sh sends a failure alert at 8am if no file is found. v1.0.0
